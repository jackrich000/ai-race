- [x] **Pipeline health checks** (PR #48, merged 2026-05-07) — Three complementary defenses:
  - **Source-level abort thresholds** in `scripts/update-data.js`. AA: 100, SWE: 30, ARC: 50, Epoch: 100. If a fetcher returns below threshold, ingestion aborts before any DB writes; existing `benchmark_scores` is preserved. Pipeline issue gets a "Source Health Failure" section. Healthy runs render a "Source Health" table in the issue body for ongoing trend visibility.
  - **Scraped-article streak alert** via new `pipeline_runs` table. Tracks `articles_scraped` and `scores_yielded` per lab per run. Alerts when 4 consecutive runs show either no_articles (scraper broken) or no_scores (extraction prompt / page template drift). Won't fire until 4 historical runs exist (~4 weeks).
  - **Migration runner** (`scripts/apply-migrations.mjs`, `npm run migrate`). Applies `migrations/*.sql` via the Supabase Management API using a Personal Access Token. SHA-256 hash tracking detects content drift; transactions wrap each migration for atomic rollback. Stays manual for now (chose option A — automating in CI deferred until migrations become more frequent). Known limitation: hash check is line-ending sensitive; if `npm run migrate` is ever run from a non-Windows checkout, it'll spuriously claim all migrations have been edited. Fix when this becomes relevant.
